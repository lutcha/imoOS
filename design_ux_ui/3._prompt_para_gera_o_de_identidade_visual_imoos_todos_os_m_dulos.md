### Inputs & Form Controls
```

- Border: 1px solid gray-300, rounded-md
- Focus: ring-2 ring-primary-500, border-primary-500
- Error: border-error, text-error
- Disabled: bg-gray-100, cursor-not-allowed

Select/Dropdown:
- Same as input + chevron icon right
- Menu: shadow-lg, bg-white, rounded-md

Checkbox/Radio:
- 20x20px, rounded-sm/rounded-full
- Checked: bg-primary-600, white checkmark
- Focus: ring-2 ring-primary-500

Date Picker:
- Input + calendar icon
- Calendar popup: grid 7 columns, highlighted today, selected range
```

### Data Display Components
```
Card:
- bg-white, rounded-lg, shadow, p-6
- Header: title + actions dropdown
- Content: flexible
- Footer: optional actions

Table:
- Sticky header, zebra striping optional
- Sortable columns (chevron indicator)
- Row hover: bg-gray-50
- Selected row: bg-primary-50, border-l-4 border-primary-600
- Pagination: bottom right, page numbers + prev/next

Badge/Status Pill:
- Small, rounded-full, px-2.5 py-0.5, text-xs font-medium
- Colors: success (green), warning (yellow), error (red), neutral (gray), info (blue)

Empty State:
- Icon (illustration), title, description, CTA button
- Centered, generous padding
```

### Feedback Components
```
Alert:
- Variants: info, success, warning, error
- Layout: icon + title + description + dismiss button
- Border-left colored by variant

Toast:
- Position: top-right, slide-in animation
- Auto-dismiss: 5s (configurable)
- Actions: optional button + dismiss

Modal/Dialog:
- Overlay: bg-black/50, fixed inset-0
- Content: bg-white, rounded-lg, shadow-xl, max-w-lg, centered
- Header: title + close button
- Body: content area
- Footer: actions (cancel + confirm)

Tooltip:
- Small, dark bg, white text, rounded, arrow pointer
- Trigger: hover or focus
```

### Navigation Components
```
Sidebar:
- Fixed left, w-64, bg-white, border-r
- Logo top, nav items with icons, user profile bottom
- Collapsible: w-20, icons only

Topbar:
- Fixed top, h-16, bg-white, border-b, px-6

Tabs:
- Horizontal: border-b, active: border-b-2 border-primary-600, text-primary-600

Breadcrumb:
- Text-sm, separator: "/", last item: font-medium, no link

## 🔄 ESTADOS E INTERAÇÕES — PADRÕES GLOBAIS

### Loading States
```
- Skeleton screens para conteúdo dinâmico
- Spinner para ações pontuais (botões)
- Progress bar para operações longas (import, export)
- Placeholder text para inputs em carregamento
```

### Empty States
```
- Ilustração contextual (não genérica)
- Título claro: "Ainda não há [item]"
- Descrição: porquê e como resolver
- CTA primário: ação para criar primeiro item
- CTA secundário: aprender mais / ver exemplo
```

### Error States
```
- Inline errors: abaixo do campo, ícone + mensagem específica
- Form-level errors: banner no topo com lista de campos com erro
- System errors: modal com código de erro, mensagem amigável, botão retry
- 404/403 pages: ilustração, mensagem, link para dashboard
```

### Offline States (Web)
```
- Banner no topo: "Modo offline — algumas funcionalidades limitadas"
- Indicador em ações que requerem rede: ícone + tooltip
- Queue indicator: "3 ações pendentes de sincronização"
- Auto-reconnect: toast quando conexão restaurada
```

### Permission States
```
- Botões desativados com tooltip explicativo: "Sem permissão para [ação]"
- Conteúdo sensível: masked ou hidden com placeholder
- Upgrade prompt: "Esta funcionalidade está disponível no plano Pro"
```

## 📱 RESPONSIVIDADE — BREAKPOINTS

```css
/* Mobile First Approach */
--breakpoint-sm: 640px;   /* Tablet portrait */
--breakpoint-md: 768px;   /* Tablet landscape */
--breakpoint-lg: 1024px;  /* Desktop small */
--breakpoint-xl: 1280px;  /* Desktop standard */
--breakpoint-2xl: 1536px; /* Desktop large */

/* Adaptações por breakpoint: */
- Sidebar: full → collapsible → hidden (hamburger menu)
- Tables: horizontal scroll → card layout → stacked
- Forms: single column → multi-column
- Charts: interactive → simplified → static image
- Density: comfortable → compact → ultra-compact
```
