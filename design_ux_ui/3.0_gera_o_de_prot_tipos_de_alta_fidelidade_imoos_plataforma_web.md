Contexto do Projeto

**Nome do Produto:** ImoOS — Sistema Operativo para Promotoras e Construtoras  
**Tipo de Plataforma:** SaaS B2B Multi-Tenant Web Application  
**Público-Alvo:** 
- Gestores de Promotoras/Construtoras
- Equipas Comerciais e de Vendas
- Engenheiros e Fiscais de Obra (backoffice)
- Investidores e Stakeholders financeiros
- Administradores de Sistema

**Stack de Implementação:** Next.js + React + Tailwind CSS + Django Backend  
**Idioma:** Português (pt-CV), preparado para i18n (pt-AO, fr-SN)  
**Resolução Base:** 1440px desktop, responsivo para tablet (768px+)  
**Orientação:** Landscape (primária)

---

## **Princípios de Design (Core Values)**

1. **Clareza sobre Complexidade:** Dados densos devem ser apresentados de forma escaneável e hierárquica.
2. **Ação Orientada:** Cada screen deve ter 1-3 ações primárias claras e visíveis.
3. **Confiança Visual:** Design profissional, limpo, que transmita segurança para decisões financeiras.
4. **Consistência Multi-Tenant:** Interface deve adaptar-se ao branding de cada empresa cliente (white-label ready).
5. **Performance Percebida:** Skeletons, loading states otimizados, feedback imediato para todas as ações.
6. **Acessibilidade:** WCAG 2.1 AA minimum, contraste adequado, navegação por teclado.

## **Requisitos Técnicos para o Design (Figma)**

1. **Auto Layout em Tudo:** Todos os componentes devem usar Auto Layout para responsividade
2. **Variants Systematic:** Criar variants para todos os estados (default, hover, active, disabled, loading, error)
3. **Component Properties:** Usar boolean/text/instance swap properties para flexibilidade
4. **Design Tokens como Styles:** Definir cores, tipografia, efeitos como Figma Styles para consistência
5. **Responsive Constraints:** Definir constraints para redimensionamento (desktop → tablet)
6. **Prototype Connections:** 
   - Smart Animate para transições suaves
   - After delay para loading states
   - Conditional navigation para fluxos complexos
7. **Annotations Layer:** Camada separada com notas para desenvolvedores (API endpoints, validation rules, edge cases)
8. **Asset Export Settings:** Configurar export para SVG (ícones), PNG@2x/@3x (imagens), JSON (design tokens se possível)

## **Notas Especiais para a IA de Design / Equipa**

🎯 **Foco em Densidade de Informação:** Esta é uma ferramenta profissional. Priorizar capacidade de apresentar muitos dados de forma clara sobre minimalismo extremo.

🔄 **Feedback de Sistema:** Cada ação do utilizador deve ter confirmação visual (toast, badge update, skeleton → content).

♿ **Acessibilidade Não é Opcional:** 
- Contrast ratio mínimo 4.5:1 para texto normal
- Focus states visíveis para navegação por teclado
- ARIA labels para ícones sem texto
- Suporte a screen readers na estrutura de componentes

🌍 **Preparar para Internacionalização:** 
- Deixar espaço para textos +30% mais longos (traduções)
- Não hardcodar ícones com significado cultural específico
- Datas e moedas devem ser apresentadas via tokens, não texto fixo

📱 **Mobile Responsiveness (Tablet):** 
- Embora seja desktop-first, a sidebar deve colapsar em <1024px
- Tables devem ter scroll horizontal ou transformar-se em cards em tablet
- Touch targets manter 44px mínimo mesmo em desktop (para consistência com mobile app)

🔐 **Estados de Permissão:** 
- Mostrar como a interface muda para diferentes roles (ex: Vendedor não vê botão "Aprovar Orçamento")
- Usar layer visibility ou component variants para demonstrar RBAC
