```markdown
---
name: tenant-context
description: Generate Django code with proper multi-tenant isolation using django-tenants patterns
argument-hint: [model-name] [operation]
allowed-tools: Read, Write, Grep
---

# Tenant Context Pattern for ImoOS

## Core Rule
Every database query that accesses TENANT_APPS models MUST run inside `tenant_context()` when outside request middleware.

## ✅ Correct Pattern
```python
from django_tenants.utils import tenant_context
from apps.tenants.models import Client

def process_unit(unit_id: int, tenant_id: int):
    tenant = Client.objects.get(id=tenant_id)  # Public schema
    with tenant_context(tenant):                # Switch to tenant schema
        unit = Unit.objects.get(id=unit_id)     # ✅ Isolated query
        # ... business logic ...
```

## ❌ Dangerous Pattern
```python
# Missing tenant_context — could access wrong tenant's data!
def process_unit(unit_id: int):
    unit = Unit.objects.get(id=unit_id)  # ⚠️ Which schema?
```

## In Celery Tasks
```python
@shared_task
def my_task(tenant_id: int, unit_id: int):  # Pass IDs only
    tenant = Client.objects.get(id=tenant_id)  # Fetch from public
    with tenant_context(tenant):               # Switch schema
        # All queries now tenant-scoped
        unit = Unit.objects.get(id=unit_id)
```

## In API Views
```python
# django-tenants middleware sets connection.tenant automatically
# So views don't need explicit tenant_context — but tests do!
class UnitViewSet(viewsets.ModelViewSet):
    permission_classes = [IsAuthenticated, IsTenantMember]
    queryset = Unit.objects.all()  # ✅ Auto-scoped by middleware
```

## Testing Reminder
Always wrap test queries in tenant_context:
```python
def test_unit_isolation(tenant_a, tenant_b):
    with tenant_context(tenant_a):
        Unit.objects.create(...)  # Goes to schema_a
    with tenant_context(tenant_b):
        assert Unit.objects.count() == 0  # schema_b is empty
```

argument-hint: [migration-type] [tenant-scope]

# Multi-Tenant Migration Patterns for ImoOS

## Migration Types
1. **Shared migrations** (public schema): Client, Domain, auth tables
   ```bash
   python manage.py migrate_schemas --shared
   ```

2. **Tenant migrations** (all schemas): business models (Project, Unit, etc.)
   ```bash
   python manage.py migrate_schemas
   ```

3. **Single-tenant migration** (testing/enterprise):
   ```bash
   python manage.py migrate_schemas --schema=empresa_a
   ```

## Creating Migrations
```bash
# Standard: affects all tenants
python manage.py makemigrations inventory
python manage.py migrate_schemas

# Shared-only: auth/tenant infrastructure
python manage.py makemigrations tenants
python manage.py migrate_schemas --shared
```

## Rollout Strategy for Large Tenants
```python
# apps/core/management/commands/rollout_migration.py
from django.core.management.base import BaseCommand
from django_tenants.utils import get_tenant_model, schema_context

class Command(BaseCommand):
    help = "Roll out migration to tenants in batches"
    
    def add_arguments(self, parser):
        parser.add_argument('--batch-size', type=int, default=10)
        parser.add_argument('--delay-seconds', type=int, default=60)
    
    def handle(self, *args, **options):
        Tenant = get_tenant_model()
        tenants = Tenant.objects.filter(is_active=True).order_by('id')
        
        for i, tenant in enumerate(tenants):
            with schema_context(tenant.schema_name):
                # Run migration logic here
                self.stdout.write(f"Migrated {tenant.schema_name}")
            
            # Rate limit to avoid DB overload
            if (i + 1) % options['batch_size'] == 0:
                time.sleep(options['delay_seconds'])
```

## Rollback Procedure
```bash
# 1. Stop Celery workers to prevent new writes
# 2. Backup affected schemas
pg_dump -h localhost -U imos -n empresa_a imos > backup_empresa_a.sql

# 3. Rollback migration
python manage.py migrate_schemas inventory 0005_previous_migration

# 4. Verify data integrity
python manage.py shell --schema=empresa_a -c "from apps.inventory.models import Unit; print(Unit.objects.count())"

# 5. Restart workers
```
